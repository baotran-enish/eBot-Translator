1. GENERAL TRANSLATION PRINCIPLES & CORE RULES
--------------------------------------------------------------------------------
[Rule 1: Technical & Industry Terminology]
- DO NOT translate standard game development, engineering, or 3D art technical 
  terms into pure Vietnamese if the English term is the industry standard.
- ALWAYS prioritize the English equivalent or its standard abbreviation.
- Example: 
  * JP: 衝突判定 (Shoutotsu Hantei) -> VN: Hitbox / Collision (NOT: Phán đoán va chạm)
  * JP: 描画 (Byouga) -> VN: Render / Rendering (NOT: Vẽ / Miêu họa)

[Rule 2: Tone and Voice]
- Professional, technical, precise, and respectful yet modern.
- Maintain an industry-insider tone. Avoid sounding like an academic dictionary 
  or an outsider translating literally.
- Use a neutral, collaborative tone appropriate for cross-border B2B and 
  internal production communications.

[Rule 3: Pronouns & Formality Constraints]
- When writing documentation or internal communication guides:
  * Use "Chúng tôi" / "Team" / "Dự án" for the company/team side.
  * Use "Bạn" / "Developer" / "Artist" (or specific role) to address the reader.
- Avoid informal or overly casual Vietnamese pronouns (cậu, tớ, mình) unless 
  explicitly creating a casual community/CS facing guide.
- Keep sentences concise. Japanese technical writing often strings multiple 
  nouns together; break them down into functional technical English/Vietnamese 
  phrases.

[Rule 4: Text-Formatting & Encoding]
- File must remain strictly UTF-8 encoded to preserve Japanese kanji and 
  Vietnamese diacritics.
- Place brackets around UI elements, variables, or keys where applicable: [OK], [Cancel], {user_name}.


2. SYSTEMATIC GRAMMAR & STYLISTIC MAPPING
--------------------------------------------------------------------------------
- JP: ~を実装する (Jisou suru) 
  -> VN: Implement [Feature/Function] / Triển khai [Tính năng]
- JP: ~を修正する (Shuusei suru) 
  -> VN: Fix [Bug/Issue] / Sửa đổi [Yếu tố]
- JP: ~を共通化する (Kyuutuuka suru)
  -> VN: Refactor thành component dùng chung / Commonize (Engineering context)
- JP: 仕様 (Shiyou)
  -> VN: Spec / Specification (NOT: Thông số kỹ thuật)
- JP: 不具合 / バグ (Fuguai / Bagu)
  -> VN: Bug / Issue (NOT: Lỗi hỏng)
- JP: 実機確認 (Jikki kakunin)
  -> VN: Test trên thiết bị thật / On-device testing

4. ANIME-SPECIFIC & STYLISTIC TERMINOLOGY
--------------------------------------------------------------------------------
Because this is an *Anime Game Company*, general gaming terms must be carefully 
aligned with Otaku/Anime subculture terminology standard in both Japan and Vietnam.

- JP: 立ち絵 (Tachie)
  -> VN: Standing artwork / Tachie (NOT: Ảnh đứng)
- JP: スチル / 一枚絵 (Suchiru / Ichimaie)
  -> VN: CG / Key visual / Still illustration (Hình minh họa sự kiện cốt truyện)
- JP: デフォルメ / ちびキャラ (Deforume / Chibi)
  -> VN: Chibi / Deformed character
- JP: ボイス / CV (Character Voice)
  -> VN: VA (Voice Actor) / CV / Seiyuu (Giữ nguyên CV/Seiyuu nếu là văn hóa Nhật)
- JP: 属性 (Zokusei)
  -> VN: Element / Attribute (Hệ/Thuộc tính của nhân vật, ví dụ: Hỏa, Thủy, Phong)
- JP: 限界突破 (Genkai Toppa)
  -> VN: Limit Break (NOT: Vượt qua giới hạn)
- JP: 覚醒 (Kakusei)
  -> VN: Awaken / Awakening (Thức tỉnh nhân vật)
- JP: 進化 (Shinka)
  -> VN: Evolve / Evolution (Tăng bậc / Tiến hóa)


5. CONSTRAINTS & FORBIDDEN TRANSLATIONS (BLACK LIST)
--------------------------------------------------------------------------------
To prevent awkward, literal Vietnamese translations, strictly avoid the following:

❌ FORBIDDEN: ハンター (Hantaa) -> Thợ săn (If it's an in-game technical class name, keep "Hunter").
❌ FORBIDDEN: スキルを打つ (Sukiru wo utsu) -> Đánh kỹ năng (Use: "Cast skill" or "Sử dụng kỹ năng").
❌ FORBIDDEN: バグを殺す (Bagu wo korosu) -> Giết bug (Use: "Fix bug" or "Sửa lỗi").
❌ FORBIDDEN: ユーザー (Yuuzaa) -> Người sử dụng (Use: "User" or "Người chơi" depending on internal/external context).
❌ FORBIDDEN: キャラクター (Kyarakutaa) -> Nhân vật hoạt hình (Use: "Character" or "Char").


6. CONTEXT-BASED VARIATION EXAMPLES
--------------------------------------------------------------------------------
The word [運用 - Unyou] changes completely based on target audience:
- If context is Engineering (Server/DevOps): -> VN: Operation / Deployment (Vận hành hệ thống).
- If context is Game Design/Live Ops: -> VN: Live Operations / Live Ops (Vận hành sự kiện, update game).

The word [マスター - Masutaa]:
- If context is Database/Dev: -> VN: Master Data (Dữ liệu cấu hình).
- If context is Git/Version Control: -> VN: Master branch / Main branch.
- If context is Gameplay/Lore: -> VN: Master (Chủ nhân - ví dụ game Fate/Grand Order style).
